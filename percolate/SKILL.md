# /percolate

Percolate updates through all related surfaces (code, comments, docs, PR metadata, ticket trackers, memory, etc.) AS you work, not at the end. Information rot compounds quickly, leading to confusion and loss of focus. Defend against this pre-emptively by keeping all in-scope information surfaces fresh. Changes in one should generally percolate through the others as needed.
